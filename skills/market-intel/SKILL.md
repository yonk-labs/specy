# market-intel (curated for specy)

Map who else solves this and earn the differentiation.

## Method
1. **Find comparables** — `WebSearch` for ≥3 real alternatives (products, OSS projects, or the manual workaround people use instead). Name them; don't invent them.
2. **Profile each** — one line each: what they do well, where they're weak.
3. **Position** — state the one thing this product does that they don't, or do badly. That's the wedge.
4. **Stress the moat** — answer honestly: why won't a comparable copy this in a weekend? If you can't answer, the differentiation is weak — say so.

Cite every external claim with the URL it came from.

## Output
Write `evidence/market-intel.md`:
- **Comparables:** table or list of ≥3, each with strength / weakness.
- **Positioning statement:** the wedge in one sentence.
- **Moat:** why it survives a fast follower (or a note that it might not).

## Done when
≥3 comparables are named AND a differentiation that survived at least one counter-argument is recorded.
